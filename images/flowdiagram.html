Corafone API · Integration Flow
Step 01
Authenticate
Generate an HMAC-SHA256 signature using your API key and a timestamp. Include it in every request header.
X-Signature + X-Timestamp
Step 02
Create Account
POST a debtor account with contact details, balance, and client metadata. Receive an account_id.
POST /accounts
Step 03
Launch Campaign
A Corafone team member launches the campaign via the platform. Cora then schedules and manages outbound contact attempts automatically.
Via Platform
Step 04
Cora Places Calls
AI agent contacts debtors, negotiates payments, handles objections, and processes arrangements — no human involvement required.
Automated
Outcome
Call Result Recorded
Each call outcome (payment arranged, promise-to-pay, not reachable, disputed) is recorded on the account. You can retrieve call details and recordings via GET /calls/:id or receive real-time events via webhooks.
Webhook Events
call.completed
Fired when a call ends. Includes duration, disposition, transcript reference, and recording URL.
payment.arranged
Debtor agreed to a payment plan or one-time payment during the call. Includes amount and schedule.
account.updated
Account status changed — e.g. moved to promise-to-pay, paid, or do-not-contact.
All requests require HMAC-SHA256 signing. See /authentication for details.
